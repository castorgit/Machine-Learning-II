# Projects

This directory contains information about course projects.

## Course Projects

### Midterm Project (15% of grade)
A focused project demonstrating understanding of topics from the first half of the course.

**Timeline:**
- Week 5: Project proposal due
- Week 6-7: Development
- Week 8: Submission and presentation

**Requirements:**
- Apply techniques from Weeks 1-7
- Use real-world dataset
- Implement at least two different approaches
- Compare and analyze results
- 5-minute presentation

### Final Project (20% of grade)
A comprehensive project showcasing advanced ML skills and creativity.

**Timeline:**
- Week 10: Project proposal due
- Week 11-14: Development
- Week 15: Final presentation and report

**Requirements:**
- Original problem or novel approach
- Complex dataset or challenging task
- Advanced ML techniques from the course
- Thorough experimental evaluation
- Well-documented code
- 10-minute presentation
- Detailed written report

## Project Proposal Guidelines

Your proposal should include:
1. **Title and Team Members**
2. **Problem Statement** (2-3 paragraphs)
   - What problem are you solving?
   - Why is it important/interesting?
3. **Dataset** (1 paragraph)
   - What data will you use?
   - How will you obtain it?
   - Size and characteristics
4. **Approach** (2-3 paragraphs)
   - What techniques will you apply?
   - What models will you compare?
   - How will you evaluate success?
5. **Timeline** (bullet points)
   - Week-by-week plan
6. **Expected Challenges**
   - What difficulties do you anticipate?

## Evaluation Criteria

### Technical (60%)
- Problem complexity and difficulty
- Appropriate methodology
- Correct implementation
- Thorough experimentation
- Results quality

### Analysis (20%)
- Insightful results interpretation
- Comparison of approaches
- Error analysis
- Discussion of limitations

### Presentation (10%)
- Clear problem motivation
- Effective visualizations
- Good communication
- Handles questions well

### Documentation (10%)
- Well-organized code
- Clear README
- Reproducible results
- Good code quality

## Submission Requirements

### Code
- All source code in organized directory
- requirements.txt or environment.yml
- README.md with setup instructions
- Jupyter notebooks (if applicable)
- No large data files (provide download links)

### Report
- PDF format, 6-8 pages (midterm), 10-15 pages (final)
- Include: abstract, introduction, methodology, experiments, results, discussion, conclusion, references
- Use figures and tables effectively
- Cite all sources

### Presentation
- Slides (PDF or PowerPoint)
- Live demo (optional but impressive)
- Practice timing
- Be prepared for questions

## Tips for Success

### Do's ✅
- Start early and iterate
- Choose a topic you're passionate about
- Use appropriate scope - not too big, not too small
- Validate your approach with simpler experiments first
- Document as you go
- Version control everything
- Ask for feedback early
- Test on multiple datasets/conditions

### Don'ts ❌
- Don't choose overly ambitious projects
- Don't start without a clear plan
- Don't ignore baseline comparisons
- Don't fabricate or cherry-pick results
- Don't leave documentation for the end
- Don't work in isolation - get feedback
