# Contributing to Machine Learning II Course

Thank you for your interest in contributing to this course repository!

## For Students

### Submitting Assignments
1. Fork this repository to your personal GitHub account
2. Clone your fork locally
3. Create a new branch for your assignment: `git checkout -b assignment-X-yourname`
4. Complete the assignment in the appropriate directory
5. Test your code thoroughly
6. Commit your changes with clear messages
7. Push to your fork
8. Create a Pull Request to submit your work

### Assignment Submission Guidelines
- Include your name and student ID in assignment files
- Follow the provided code structure and naming conventions
- Include comments explaining your approach
- Add a README.md with:
  - How to run your code
  - Dependencies required
  - Results and observations
  - Any challenges faced

### Lab Exercises
- Complete lab notebooks in the `labs/` directory
- Save your completed notebooks with your name: `lab-X-yourname.ipynb`
- Commit and push regularly to track your progress

## For Instructors and TAs

### Adding Course Materials
1. Create a new branch: `git checkout -b feature/new-material`
2. Add materials to appropriate directories:
   - Lectures → `lectures/`
   - Assignments → `assignments/`
   - Labs → `labs/`
   - Resources → `resources/`
3. Update relevant README files
4. Test all code examples
5. Submit a Pull Request for review

### Content Guidelines
- Include learning objectives at the start of each module
- Provide clear, commented code examples
- Add visual aids (diagrams, plots) where helpful
- Include references to relevant papers/resources
- Test all notebooks before publishing

### Code Standards
- Follow PEP 8 style guide for Python code
- Use descriptive variable and function names
- Include docstrings for all functions
- Add type hints where appropriate
- Keep code modular and reusable

## General Guidelines

### File Naming Conventions
- Use lowercase with hyphens: `assignment-1-clustering.ipynb`
- Be descriptive: `cnn-image-classification.py`
- Include dates for versioned content: `lecture-2-2026-01-15.pdf`

### Documentation
- Update README files when adding new content
- Document all dependencies in requirements.txt
- Include setup instructions for complex environments
- Add comments to explain non-obvious code

### Code Quality
- Test your code before committing
- Remove debug print statements
- Clear notebook outputs before committing (for clean diffs)
- Use .gitignore to exclude large files and datasets

### Commit Messages
- Use clear, descriptive commit messages
- Start with a verb: "Add", "Update", "Fix", "Remove"
- Reference issue numbers when applicable
- Example: "Add CNN lecture materials for week 8"

## Getting Help
- Check existing issues before creating new ones
- Use clear, descriptive titles for issues
- Include error messages and system information
- Tag issues appropriately (bug, question, enhancement)

## Code of Conduct
- Be respectful and professional
- Provide constructive feedback
- Help fellow students when possible
- Report any violations to the instructor

Thank you for contributing to making this a great learning resource!
