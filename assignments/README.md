## Submission Guidelines

### How to Submit
1. Complete your assignment in the designated directory
2. Ensure all code runs without errors
3. Include a comprehensive README with:
   - Your name and student ID
   - How to run your code
   - Results and analysis
   - Challenges and solutions
4. Test your submission in a clean environment
5. Submit via Pull Request (see CONTRIBUTING.md)

### What to Include
- All source code files (.py or .ipynb)
- README.md with documentation
- requirements.txt (if using additional libraries)
- Results (plots, metrics, outputs)
- Do NOT include large data files (use links instead)

### Grading Criteria
Each assignment is graded on:
- **Correctness (50%):** Does your solution work? Are results accurate?
- **Code Quality (20%):** Is code clean, well-organized, and documented?
- **Documentation (15%):** Is your approach clearly explained?
- **Analysis (15%):** Do you interpret and discuss results?

### Late Policy
- -10% per day late (maximum 3 days)
- Contact instructor for extensions due to emergencies

## Getting Help
- Review lecture materials and examples

## Academic Integrity
- Write your own code
- Cite all external resources
- Discuss concepts, not solutions
- See course syllabus for full policy

## Tips for Success
✅ Start early - assignments take longer than expected  
✅ Test incrementally - don't write all code at once  
✅ Use version control - commit frequently  
✅ Document as you go - don't wait until the end  
✅ Validate results - sanity check your outputs  
✅ Ask questions - don't struggle alone
